## Context

需要选择运行平台和展示方式：
- 定时运行 RSS 采集
- 展示分类结果
- 控制成本

候选方案：
- 自建服务器 + 自建前端
- GitHub Actions + GitHub Pages
- Vercel/Netlify + Serverless Functions

## Decision

选择 GitHub Actions + GitHub Pages。

具体设计：
- GitHub Actions 每 6 小时定时触发
- 采集结果写入 git 仓库
- GitHub Pages 直接渲染 Markdown

## Consequences

优点：
- 免费额度充足
- 零运维
- 版本控制天然支持
- Markdown 表格直接可读

缺点：
- GitHub Actions 单次运行上限 6 小时
- cron 最小粒度 5 分钟
- 仓库体积长期增长
