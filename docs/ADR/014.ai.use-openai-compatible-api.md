## Context

ADR-008 确定了多 AI Provider 支持方案，但实现上依赖 `openai` SDK。

问题：
1. `openai` SDK 是一个较重的依赖
2. 无法直接支持其他 OpenAI 兼容 API（vLLM、Ollama、LocalAI 等）
3. OpenRouter 本质上是 OpenAI 兼容 API，却需要单独适配

## Decision

用 `requests` 直接实现 OpenAI 兼容 API 调用，去掉 `openai` SDK 依赖。

### 架构

| Provider | 实现方式 | 依赖 |
|----------|----------|------|
| openai / openrouter / custom | requests + OpenAI 兼容 API | 无额外依赖 |
| claude | anthropic SDK | anthropic |

### 配置

```toml
[ai]
provider = "openrouter"  # openai | claude | openrouter | custom
model = "nvidia/nemotron-3-nano-30b-a3b:free"
# base_url = ""  # 可选：自定义 API 端点
# api_key_env = ""  # 可选：API Key 环境变量名
```

## Consequences

优点：
- 支持任意 OpenAI 兼容 API（vLLM、Ollama、LocalAI 等）
- 减少依赖（去掉 openai SDK）
- 更灵活的 `base_url` 配置
- 代码更简单，无需维护 SDK 兼容层

缺点：
- Anthropic API 需要单独维护（非 OpenAI 兼容）
- 需要自行处理 HTTP 请求细节
