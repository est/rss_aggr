## Context

某些 RSS 源是聚合源（如 Hacker News、Reddit），包含大量链接但缺少正文内容。对这些源进行 AI 分类：
- 成本高（每篇文章都需要 API 调用）
- 效果差（缺少正文难以准确分类）
- 价值低（聚合源本身已有热度排序）

## Decision

自动检测聚合源并跳过 AI 分类。

具体设计：
- 检测已知聚合源（HN、Reddit 等）
- 聚合源文章直接输出，不经过 AI 分类
- 分类标记为 "Aggregated"

## Consequences

优点：
- 节省 AI API 成本
- 加速处理流程
- 聚合源本身已有筛选机制

缺点：
- 可能错过聚合源中的优质文章
- 需要维护聚合源列表
