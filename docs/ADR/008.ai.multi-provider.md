## Context

AI 分类是核心功能，需要选择：
- 单一 provider 还是多 provider
- 如何处理 provider 不可用的情况
- 如何控制成本

候选方案：
- 单一 OpenAI
- 多 Provider（OpenAI/Claude/OpenRouter）
- 本地模型

## Decision

支持多 AI Provider：OpenAI、Claude、OpenRouter。

具体设计：
- `config.toml` 的 `ai.provider` 选择当前使用的 provider
- 各 provider 通过环境变量配置 API Key
- 分类失败时 fallback 为 "Unclassified"，不阻断流程

## Consequences

优点：
- 灵活切换 provider
- 单一 provider 故障不影响整体
- 可根据成本/质量选择

缺点：
- 需要维护多个 provider 的适配代码
- 不同 provider 的 prompt 响应可能不一致
