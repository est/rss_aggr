## Context

对整体架构、设计和功能实现的代码审查发现。

## 发现

### Bug（需要修复）

#### 1. `feeds_state` 未传递给 `fetch_all_feeds` ✅ Fixed

**严重程度：Medium**

`src/main.py:107-113` 调用 `fetch_all_feeds` 时未传递 `feeds_state` 参数，导致 HTTP ETag/Last-Modified 条件请求完全失效，每次运行都全量抓取所有 feed。

```python
# 修复后
results = fetch_all_feeds(
    due_feeds,
    timeout=fetch_cfg.get("timeout_seconds", 15),
    keep_days=keep_days,
    user_agent=user_agent,
    skip_titles=skip_titles,
    feeds_state=state.get("feeds", {}),  # ✅ 已添加
)
```

#### 2. `future.result()` 未保护 ✅ Fixed

**严重程度：Low-Medium**

`src/fetcher.py:210-211` 中 `future.result()` 可能重新抛出异常，导致单个 feed 异常会杀死整个抓取循环。

```python
# 修复后
for future in as_completed(futures):
    try:
        r = future.result()
    except Exception as e:
        f = futures[future]
        r = {
            "feed": f,
            "status": "error",
            "error": str(e),
            "entries": [],
            "fetched_at": datetime.now(timezone.utc).isoformat(),
        }
```

### 设计问题

#### 1. 聚合器检测未生效（ADR-009 未实现）

`step_classify` 没有检查 `is_aggregator` 状态，聚合源文章仍会进入 AI 分类。

#### 2. CI 持久化机制可能不同步

`actions/cache` 和 gh-pages worktree 两种持久化机制可能在某些场景下不同步。

#### 3. CI 直接推送到 main

工作流直接 push 到 main 分支，两次运行可能冲突。

### 其他

- `aggregator.py` 注释与代码逻辑矛盾（or vs and）
- `opml_parser.py` 命名误导（实际解析 TOML）
- `skip_titles` 默认值类型标注不准确
