## Context

CI 工作流需要在两个分支间同步数据：
- `main` — 代码
- `gh-pages` — 运行时数据（output、state.json、articles.json）

原有方案存在以下问题：
1. 用 `git archive` + `tar` 恢复文件，需要处理分支不存在的情况
2. `cp -rn` 不覆盖已有文件，可能导致恢复不完整
3. 恢复逻辑分散在多处，难以维护

## Decision

统一使用 `git worktree` 管理分支切换。

具体流程：
1. Checkout main（默认 worktree）
2. 创建 gh-pages worktree
3. 从 gh-pages worktree 复制数据到当前工作目录
4. 运行 fetch/classify/cleanup
5. 复制结果回 gh-pages worktree
6. 提交并推送 gh-pages worktree
7. 清理 worktree

## Consequences

优点：
- 恢复逻辑统一，不需要 `git archive` + `tar`
- 更容易处理分支不存在的情况
- 逻辑更清晰，减少出错可能

缺点：
- 需要清理 worktree（使用 --force）
