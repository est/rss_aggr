## Context

RSS 文章分类后需要存储结果。需要决定：
- 是否允许修改已输出的文件
- 如何处理重复运行
- 如何保证历史数据完整性

## Decision

Output 不可变：`output/YYYY/MMDD.md` 一旦写入不再修改。

具体规则：
- 同一天多次运行只追加新文章（自动去重）
- 不覆盖、不删除已有内容
- 已发布的 HTML 不受影响

## Consequences

优点：
- 历史数据完整，可追溯
- 无需担心运行顺序导致数据丢失
- GitHub Pages 渲染稳定

缺点：
- 修正分类错误需要手动编辑或新建修正条目
- 文件可能因重复运行而增大（但去重机制控制）
