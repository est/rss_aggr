## Context

项目需要配置文件来管理：
- RSS 源列表（数百个源）
- AI provider 选择
- 抓取参数
- 过滤规则
- 分类列表

候选方案：
- YAML
- TOML
- JSON
- INI

## Decision

选择 TOML 作为配置格式。

## Consequences

优点：
- Python 3.11+ 内置 `tomllib`，无需额外依赖
- 语法简洁，比 YAML 更少歧义
- 支持注释
- 适合分层配置（`[[category.feed]]` 嵌套结构）

缺点：
- 生态工具不如 YAML 丰富
- 复杂嵌套时可读性下降

## References

- `config.toml` — 主配置
- `feeds.toml` — RSS 源列表
- `sources.toml` — 外部 OPML 源 URL
