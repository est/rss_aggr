## Context

现有文件职责存在以下问题：

1. **cache.json 职责不清晰** — 名为"cache"但实际存储待分类文章，与 feed 状态混淆
2. **state.json 包含 skipped_links** — 文章 skip 信息与 feed 状态混在一起
3. **output 恢复逻辑脆弱** — CI 从 gh-pages 恢复时缺少错误处理，导致文件消失
4. **写入非原子** — `write_text` 全量覆盖，进程中断可能丢失数据

## Decision

### 1. 文件重命名与职责定义

| 文件 | 存储位置 | 职责 |
|------|----------|------|
| `state.json` | gh-pages | Feed 运行状态（last_ok, failures, disabled, etag） |
| `articles.json` | gh-pages | 文章缓存（待分类文章，14 天窗口） |

### 2. 代码改动

- `src/cache.py` → 重命名为 `src/articles.py`
- `src/state.py` — 删除 `skipped_links` 相关逻辑
- `src/storage.py` — 添加原子写入（先写 .tmp 再 rename）

### 3. CI 改进

- 修复 gh-pages 恢复逻辑，明确处理分支不存在的情况
- `cache.json` → `articles.json`

## Consequences

优点：
- 职责清晰：feed 状态 vs 文章缓存
- 原子写入防止数据丢失
- CI 恢复更健壮

缺点：
- 需要迁移现有数据
- 代码改动涉及多个文件
