## Context

数百个 RSS 源中，部分可能：
- 服务器宕机
- URL 变更
- 长期不更新

需要一种机制来：
- 检测源是否还活着
- 避免浪费资源抓取失效源
- 自动恢复或禁用

## Decision

采用 HTTP HEAD 检测 + 连续失败自动禁用。

具体设计：
- 抓取前进行 HTTP HEAD 检测
- 记录连续失败次数
- 连续失败 3 次后自动禁用该源
- 禁用后不参与后续抓取

## Consequences

优点：
- 自动清理失效源
- 减少无效请求
- 保护整体系统稳定性

缺点：
- 临时故障可能导致误禁用
- 需要手动恢复误禁用的源
